# 从其他设备访问说明

## 快速开始

1. **启动服务**
   ```bash
   ./quick_start.sh
   ```
   或
   ```bash
   ./start.sh
   ```

2. **查看启动信息**
   启动脚本会自动显示本机IP地址，例如：
   ```
   🌐 其他设备访问（同一网络）:
      前端: http://192.168.1.100:3000
   ```

3. **在其他设备上访问**
   - 确保手机/电脑与服务器在同一局域网（WiFi）中
   - 在浏览器中输入显示的IP地址和端口，例如：`http://192.168.1.100:3000`

## 手动查找IP地址

如果启动脚本没有显示IP地址，可以手动查找：

### Linux
```bash
ip addr show
# 或
ifconfig
# 查找类似 192.168.x.x 或 10.x.x.x 的地址
```

### Mac
```bash
ifconfig | grep "inet "
# 查找类似 192.168.x.x 的地址（排除 127.0.0.1）
```

### Windows
```cmd
ipconfig
# 查找"IPv4 地址"，通常是 192.168.x.x
```

## 注意事项

1. **防火墙设置**
   - 确保防火墙允许端口 3000 和 8000 的入站连接
   - Linux: `sudo ufw allow 3000` 和 `sudo ufw allow 8000`
   - Mac: 系统偏好设置 → 安全性与隐私 → 防火墙
   - Windows: 控制面板 → Windows Defender 防火墙

2. **网络要求**
   - 所有设备必须在同一局域网（WiFi）中
   - 不能跨网络访问（除非配置了端口转发）

3. **安全提示**
   - 当前配置允许局域网内所有设备访问
   - 如果需要在公网访问，请配置反向代理和HTTPS

## 常见问题

**Q: 无法访问？**
- 检查防火墙设置
- 确认设备在同一WiFi网络
- 确认IP地址正确

**Q: 可以访问但上传文件失败？**
- 检查后端服务是否正常运行（端口8000）
- 查看后端日志文件 `backend.log`

**Q: 如何停止服务？**
- 在启动服务的终端按 `Ctrl+C`
